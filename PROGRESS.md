# 开发进度

## 2026-07-12 第一阶段完成

- 根据共享方案建立 ClassMind 第一阶段 MVP。
- 使用 OR-Tools CP-SAT 实现硬约束排课。
- 创建 4 位教师、3 间教室、4 个班级、4 门课程、8 个课次的演示数据。

验收目标：演示数据生成 8 个课次，硬冲突为 0，全部测试通过。

## 2026-07-12 完整项目扩展

- 增加学生体验、教师稳定、校区效率三种优化策略。
- 增加偏好满足率、容量适配率和综合评分卡。
- 增加本地 REST API 与响应式可视化驾驶舱。
- 增加候选方案切换和 AI 解释区。
- 单元测试扩展到策略组合与评分范围。
- 增加 `launch_classmind.bat`，支持双击检查环境、启动服务并打开驾驶舱。
- 修复 BAT 的 UTF-8 中文解析和嵌套引号错误；改为纯 ASCII 脚本，并加入 20 秒后端健康检查和 `stop_classmind.bat`。
- 左侧导航改为真实功能锚点；新增临时请假最小影响调课与资源概览功能区。
- 新增独立 API 集成测试，覆盖首页导航、健康检查、演示数据、三方案、通用求解和局部重排。
- 新版服务迁移到独立端口 `8766`，避免遗留的 `8765` 进程阻塞更新与验收。
- 增加独立 `run_server.bat` 服务窗口；一键启动器通过该窗口运行后端并执行30秒健康检查，消除嵌套引号错误且不依赖管理员权限。
- 前端由单页锚点改为统一侧栏母版加四个独立子页面，分别承载驾驶舱、智能排课、调课中心和资源概览。
- 智能排课页移除候选策略侧栏，升级为全宽08:00—17:30纵向时间课表，明确标出12:00—13:30午休禁排区；演示排课数据同步改为日间时段。
