ClassMind 高途智能排课脑——技术路线说明

一、项目定位

ClassMind 面向线下校区教务排课场景，目标是建立一套“零硬冲突、可解释、可调度”的智能排课系统。

二、总体架构

系统采用四层架构：

1. 前端交互层
   - 决策驾驶舱：展示核心指标和方案摘要；
   - 智能排课：展示 08:00—17:30 全日周课表；
   - 调课中心：处理教师请假与临时调整；
   - 资源概览：管理教师、教室、班级和课程数据。

2. API 与业务层
   - 提供健康检查、数据读取、课表求解和调课接口；
   - 负责方案评分、教师负载统计和课表差异分析。

3. 算法与校验层
   - 使用 Google OR-Tools CP-SAT 生成课表；
   - 使用独立校验器对求解结果进行二次复核。

4. 数据模型层
   - 将教师、教室、班级、课程和时间槽统一转换为不可变数据模型；
   - 使用 JSON 作为前后端和业务数据的交换格式。

三、核心排课算法

1. 决策变量
   - 为“课次—教师—教室—时段”组合建立布尔变量。

2. 硬约束
   - 每个课次必须且只能安排一次；
   - 教师、教室和班级不得发生时间冲突；
   - 教师资质必须匹配课程；
   - 教室容量和设备必须满足课程需求；
   - 课程不得安排在不可用时间和午休时段。

3. 软目标
   - 提高学生时段偏好满足率；
   - 提高教师偏好满足率；
   - 减少教室容量浪费；
   - 提高课表紧凑度和资源利用率。

四、临时调课路线

教师请假时，系统按以下步骤处理：

1. 保存原课表作为基线；
2. 加入教师新的不可用时间；
3. 锁定未受影响课程；
4. 以“课程变更数量最少”为最高优先级重新求解；
5. 输出新课表、变更课次和受影响班级。

五、主要技术栈

- 后端：Python、dataclass、ThreadingHTTPServer；
- 求解器：Google OR-Tools CP-SAT；
- 前端：原生 HTML、CSS、JavaScript；
- 数据：JSON；
- 测试：unittest、API 集成测试；
- 部署：Windows BAT 一键启动，本地端口 8766 运行。

六、技术成果

- 生成的课表硬冲突数严格为 0；
- 支持全日课表、资源查看和最小影响调课；
- 前后端解耦，无需 Node.js 构建；
- 支持本地运行、测试验证和后续飞书生态接入。
